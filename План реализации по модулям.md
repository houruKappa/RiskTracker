# План реализации RiskTracker — детальная разбивка по модулям и этапам

## Условные обозначения

- **[B]** — Backend (Go)
- **[F]** — Frontend (Next.js)
- **[I]** — Инфраструктура
- **[D]** — База данных / миграции
- **[T]** — Тесты

Каждый этап содержит **контрольные метрики** — условия, при которых этап считается закрытым. Метрики делятся на три типа:
- **Функциональные** — конкретный сценарий, который должен работать
- **Кодовая база** — что должно быть в репозитории
- **Автоматическая проверка** — тесты / линтеры / сборка
