    + swagger


Описание:
     Разработка системы отслеживания, создания и оценки рисков. Роловая модель( Администратор и обычный пользователь). Создание, ведение, изменение и логгирование.Функции пользователей (
        - Обычный пользователь: Создание-редактирование рисков. Просмотр отчетов, которые ему доступны( отчеты по рискам, к которым есть доступ).
        - Администратор: Создание, изменение, просмотр истории изменения(логгирование), просмотр отчетов по фильтрам и отделам для рисков. Создание, изменение паролей пользователей, назначение ролей.
     )

Структура Риска:
Риск - событие и описание этого события (
    Секция 1 ( Шапка риска)
    - status (В работе/ завершен)
    - ID
    - Имя риска
    - Объект риска (Объект риска (ИТ-система / Проект / Процесс / Risk object (IT-system/ Project/ Process) )
    - Владелец риска ( ФИО / Имя - логин  )

    Секция 2 ( Тело риска )

    Риск разделяется на 2 группы
    | Причины|                                |Последствия|

    Для обеих групп по 3 поля( Имя, Описание, Вероятность(вероятность это относительная величина( low, medium, high, critical) ) ), то есть у причины свой список элементов с этими 3 полями, и у последствий свой список со своими элементами из этий полей.

    В конце этого списка определяется максимальная из вероятностей для причин и для последствий по отдельности

    Секция 3 (Оценка риска)

    - Вероятность наступления риска(low medium ...)
    - Влияние риска ( low, medium ...)
    - Опциональные факторы(
        - Оценка финансовых потерь
        - Оценка репутационных потерь
        - Законодательные последствия ( степени )
    - Комментарий к оценке
    )

)

У каждого риска для причин и последствий по отдельности существуют Контрмеры ( 1 риск много контрмер).
Структура контрмеры:
    header:
    - К какому риску относится( id или что то такое), к причинам или к последствиям
    Body:
    - Описание
    - Ответвенный (указание id или почты или чего то и автоподставление данных этого человека)
    - Срок выполнения ( Время создания  < Срока)


Формирование отчета:
    Автоформирование по фильтрам( по области (Объект риска из секции 1 риска)), по ответственному лицу, .....   )

    Содержание отчета:
        Для примера опишу при поиске по отв лицу, но в остальных случаях примерно то же самое тоже будет

        Поиск по ответсвенному лицу : ...
        Риски в которых он фигурирует, данные / просмотр самих рисков, сроки выполнения( в том числе сгоревших), просмотр контрмер.
        Статистика в этих рисках Этого лица(
            количество expired
            осташиеся сроки
            ...

        )

    Настроить систему нормальной полной фильтрации(по параметрам, тому подобное), в том числе завершенных рисков. Завершенные риски не удаляются из системы, они остаются со статусом и могут быть просмотрены, включены в статистику

    Должна быть воможность быстрого изменения риска со страницы отчета ( я не знаю как это реализовать, возможно как модальное окно, или другими способами)



ЛОГГИРОВАНИЕ:
    Действия пользователей логгируются. При изменении какой либо части риска или контрмеры создается запись, которая отражает время, пользователя который что то сделал, предыдущее значение и значение, на которое он изменил. Это отражается как в отчете, так и в dashboard, так и может быть экспортировано( или скопировано*). Также необходимо чтобы это логирование отражало весь жизненный цикл( по этим логам можно узнать и вернуть( вручную ) к состоянию до какого то момента( по действиям ))
